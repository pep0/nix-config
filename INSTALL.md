# Installation

This config supports two paths:

- **Fresh install** — booting the NixOS installer ISO onto a blank machine.
- **Migration** — adopting this flake on a machine already running NixOS.

Pick the one that matches your situation. Both end with `make system`
taking over the machine.

## Edit before installing

Most of these have placeholder values that you need to swap for your own.

- `hosts/<name>/default.nix` — `networking.hostName` (rename to whatever
  the machine actually is) and `system.stateVersion` (set to the release
  you first install with; **never change** post-install).
- `modules/system/users.nix` — username (also update `flake.nix`'s
  `home-manager.users.pep0` and `modules/home/default.nix`'s
  `home.username` / `homeDirectory` to match).
- `modules/system/nix.nix` — `programs.nh.flake` path (where you'll
  clone the repo to on the target machine).
- `modules/system/locale.nix` — timezone.
- `modules/home/git.nix` — name and email.
- `modules/system/stylix.nix` — replace the placeholder wallpaper with
  your own image; pick a different `base16Scheme` if you don't want
  Tokyo Night.
- `.sops.yaml` — replace the placeholder age public key with one you
  generated. See [SECRETS.md](SECRETS.md) for the bootstrap.
- `hosts/<name>/hardware-configuration.nix` — generated by
  `nixos-generate-config` during install. Drop it in the host
  directory you're targeting.
- LUKS in `modules/system/boot.nix` — uncomment and fill the UUID, or
  leave commented and let `hardware-configuration.nix` own it (the
  default).

## Path A: Fresh install

1. Build the **nixos-unstable** installer ISO onto a USB stick:

   ```
   sudo dd if=nixos-minimal-25.11pre-x86_64-linux.iso of=/dev/sdX bs=4M status=progress conv=fsync
   ```

   Replace `/dev/sdX` with the USB device (not a partition).

1. Boot from USB. On the **MacBook**, hold ⌥ (Option/Alt) at power-on
   and pick "EFI Boot". On a regular x86 box, use the firmware's boot
   menu (typically F12).

1. Get networking inside the installer.

   - **ThinkPad** — Wi-Fi works out of the box.
   - **MacBook** — the BCM4360 Wi-Fi card needs the proprietary
     `broadcom-sta` driver, which isn't in the live ISO. Either
     USB-tether your phone (Android USB tethering / iPhone Personal
     Hotspot via USB) or use a USB/Thunderbolt Ethernet adapter.

   Verify with `ping 1.1.1.1`.

1. Partition. The example below is GPT + EFI + LUKS + ext4. Adjust
   the device name (`/dev/nvme0n1` is typical for both targets).

   ```
   parted /dev/nvme0n1 -- mklabel gpt
   parted /dev/nvme0n1 -- mkpart ESP fat32 1MiB 512MiB
   parted /dev/nvme0n1 -- set 1 esp on
   parted /dev/nvme0n1 -- mkpart primary 512MiB 100%

   cryptsetup luksFormat /dev/nvme0n1p2
   cryptsetup open /dev/nvme0n1p2 cryptroot
   mkfs.ext4 -L nixos /dev/mapper/cryptroot
   mkfs.fat -F32 -n boot /dev/nvme0n1p1

   mount /dev/disk/by-label/nixos /mnt
   mkdir -p /mnt/boot && mount /dev/disk/by-label/boot /mnt/boot
   ```

   Skip the `cryptsetup` lines and `mkfs.ext4` directly on `/dev/nvme0n1p2`
   if you don't want full-disk encryption.

1. Generate hardware config:

   ```
   nixos-generate-config --root /mnt
   ```

1. Clone this repo and wire in the hardware config:

   ```
   nix-shell -p git --run 'git clone https://github.com/pep0/nix-config /mnt/etc/nixos/nix-config'
   cp /mnt/etc/nixos/hardware-configuration.nix /mnt/etc/nixos/nix-config/hosts/<name>/
   ```

   Replace `<name>` with `default` (ThinkPad) or `macbook` (MacBook).

1. Install:

   ```
   nixos-install --flake /mnt/etc/nixos/nix-config#<name>
   ```

   Set the root password when prompted, then `reboot` and pull the USB.

1. First boot — set the user password and log in:

   ```
   passwd pep0          # as root, on TTY1
   ```

   Then log out, pick `Hyprland` in the greetd login, and log back in
   as `pep0`.

## Path B: Migrating from an existing NixOS install

You skip about 80% of the fresh-install steps — partitioning,
filesystems, LUKS, the installer ISO, the bootloader. They all stay
exactly as your existing install set them up.

1. From the existing system, capture three things:

   ```
   grep -E 'stateVersion|hostName' /etc/nixos/configuration.nix
   whoami
   ```

   You need:
   - the existing `system.stateVersion` (must not change),
   - the existing hostname (you'll either rename the host or rename
     the flake entry),
   - the existing username (decide whether to adopt `pep0` or rename
     the flake's user to your existing one).

1. Clone this repo:

   ```
   nix-shell -p git --run 'git clone https://github.com/pep0/nix-config ~/nix-config'
   cd ~/nix-config
   ```

1. Reuse your existing hardware config:

   ```
   sudo cp /etc/nixos/hardware-configuration.nix hosts/<name>/
   ```

1. Reconcile differences in `hosts/<name>/default.nix`:
   - `system.stateVersion` → match the value from step 1.
   - `networking.hostName` → match your existing hostname.

   If your username is not `pep0`, sed-replace `pep0` to your real
   username across `modules/system/users.nix`,
   `modules/home/default.nix`, `modules/home/git.nix`, and
   `flake.nix` (`home-manager.users.pep0`). Keeping the username
   preserves your existing `/home`.

1. Test-build *without activating*:

   ```
   sudo nixos-rebuild build --flake ~/nix-config#<name>
   ```

   If this fails, nothing has changed on the system — read the
   error, fix, retry.

1. Switch:

   ```
   sudo nixos-rebuild switch --flake ~/nix-config#<name>
   ```

   Note: `nh` isn't installed on your system before the first switch,
   so the bootstrap step has to use stock `nixos-rebuild`. After this,
   `make system` works.

1. Move the clone to its declared home (`programs.nh.flake` in
   `modules/system/nix.nix` points at `/home/pep0/nix-config`):

   ```
   mv ~/nix-config /home/pep0/nix-config
   ```

   Or update `programs.nh.flake` to wherever you actually keep it.

1. Reboot. Some kernel modules (notably `wl` for Broadcom Wi-Fi on
   the MacBook) only take effect after a clean boot.

## Per-host notes

### `default` — ThinkPad P14s Gen 5

- After the first successful install, follow [SECUREBOOT.md](SECUREBOOT.md)
  to enable Lanzaboote (Secure Boot with your own keys).
- The NVIDIA dGPU is off by default via PRIME-offload; processes hit
  it only when launched with `nvidia-offload`. A "battery-saver"
  generation in the boot menu turns the dGPU off entirely.

### `macbook` — MacBook Pro Mid 2014, 13" (MacBookPro11,1)

- **Wi-Fi during the installer ISO** — `broadcom-sta` isn't shipped
  in the live image. Tether or use Ethernet (see step 3 above).
  Once installed, `wl` is built against the kernel and Wi-Fi works
  at boot.
- **No Lanzaboote** — Apple firmware doesn't reliably enroll
  user-supplied Secure Boot keys. This host stays on `systemd-boot`.
- **EFI partition size** — older NixOS installs sometimes created a
  256MB `/boot`. With 10 generations retained, that fills up.
  Check `df -h /boot` and either lower
  `boot.loader.systemd-boot.configurationLimit` or resize.
- **FaceTime HD camera** — non-functional. The `bcwc-pcie` driver
  has been broken on modern kernels for years.
- **Function keys** — Apple's keyboard treats `F1`-`F12` as media
  keys by default. To flip the behavior, add to the host file:

  ```nix
  boot.extraModprobeConfig = ''
    options hid_apple fnmode=2
  '';
  ```

## Sops-nix bootstrap (any host, optional)

Until you actually declare a secret, sops-nix is inert. When you
need one:

```
sudo mkdir -p /var/lib/sops-nix
sudo age-keygen -o /var/lib/sops-nix/key.txt
sudo chmod 600 /var/lib/sops-nix/key.txt
```

Take the `# public key: age1...` line from that file, paste it into
`.sops.yaml` (replacing `age1placeholder...`), commit, push. Full
walkthrough in [SECRETS.md](SECRETS.md).
